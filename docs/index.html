Settings

All values are stored in your browser's local storage and never sent anywhere except GitHub's API.

📦 GitHub repository
The owner/repo slug for your private Section 11 repo. Example: yourname/section-11
This is where your training JSON files live and where the sync workflow runs.
🔑 GitHub tokens
You can use the same token for both roles, or separate tokens with minimal permissions. Tokens are stored only in your browser local storage — never transmitted to any server other than GitHub.
Used to fetch raw files from your private repo (latest.json, intervals.json, history.json).
Minimum scopes: repo (or contents: read for fine-grained tokens).
Used to call POST /repos/{owner}/{repo}/actions/workflows/sync.yml/dispatches to start the sync.
Minimum scopes: repo or fine-grained actions: write.
⚙ Workflow
The filename of your GitHub Actions workflow. Defaults to sync.yml. Only change this if your workflow file has a different name.
The branch to dispatch the workflow against. Defaults to main.
🤖 AI tools
Add one or more AI chat tools. Each will appear as a shortcut button on the Actions page. Label is optional — URL will be used if omitted.
💾 Export / Import settings

Export your settings encrypted with a password of your choice — uses AES-256-GCM via the browser's built-in Web Crypto API. No library, nothing leaves your device. Store the file anywhere; only the password unlocks it.
